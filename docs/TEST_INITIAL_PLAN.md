## 🎯 Objetivo do Teste

Verificar se usuários que instalam o app mid-week (ex: terça-feira) recebem um plano semanal imediatamente, sem precisar esperar até segunda-feira.

---

## 📋 Cenários de Teste

### Cenário 1: Novo Usuário (Terça-feira)

**Passos**:
1. Criar novo usuário no app
2. Fazer primeiro speaking task (para criar skill profile)
3. Ir para Tab 2 (Plano)
4. Observar comportamento

**Resultado Esperado**:
- ✅ Tela mostra loading por 2-5 segundos
- ✅ Plano semanal aparece com:
  * Primary Focus (ex: "Grammar")
  * Secondary Focus (ex: "Vocabulary")
  * Período da semana (ex: "Feb 24 - Mar 2")
  * Today's Task
  * Upcoming Tasks

**Resultado Anterior (Problema)**:
- ❌ Tela mostrava: "Your weekly plan will be generated soon"
- ❌ Usuário ficava sem plano até segunda-feira

---

### Cenário 2: Usuário Sem Speaking Task

**Passos**:
1. Criar novo usuário no app
2. NÃO fazer speaking task
3. Ir para Tab 2 (Plano)

**Resultado Esperado**:
- ✅ Tela mostra: "Your weekly plan will be generated soon"
- ✅ Mensagem explicativa

**Motivo**: Sem speaking task, não há skill profile, então não há dados para gerar plano.

---

### Cenário 3: Usuário Existente (Segunda-feira)

**Passos**:
1. Usar usuário que já tem plano
2. Aguardar segunda-feira 00:00 (ou simular)
3. Abrir app e ir para Tab 2

**Resultado Esperado**:
- ✅ Plano é regenerado automaticamente (se necessário)
- ✅ Novo plano aparece baseado em mudanças de FI ou skills

### 3. Verificar snapshot criado

```sql
SELECT 
  id,
  user_id,
  week_start,
  fluency_index,
  primary_skill,
  secondary_skill,
  created_at
FROM weekly_skill_snapshot
WHERE user_id = 'user-uuid'
ORDER BY week_start DESC
LIMIT 1;
```

**Esperado**: Snapshot da semana atual existe

---

### 4. Verificar plano gerado

```sql
SELECT 
  id,
  user_id,
  week_start,
  primary_focus,
  secondary_focus,
  status,
  logic,
  created_at
FROM weekly_plans
WHERE user_id = 'user-uuid'
  AND status = 'active'
ORDER BY week_start DESC
LIMIT 1;
```

**Esperado**: Plano ativo com primary_focus e secondary_focus preenchidos

---

### 5. Verificar tasks geradas

```sql
SELECT 
  id,
  plan_id,
  day_of_week,
  skill,
  content->>'title' as title,
  difficulty,
  completed
FROM daily_tasks
WHERE plan_id = (
  SELECT id FROM weekly_plans 
  WHERE user_id = 'user-uuid' 
    AND status = 'active'
  LIMIT 1
)
ORDER BY day_of_week;
```

**Esperado**: 7 tasks (uma para cada dia da semana)

## 🐛 Troubleshooting

### Problema: Plano não é gerado

**Verificar**:
1. Edge function está deployada?
   ```bash
   supabase functions list
   ```

2. Função SQL existe?
   ```sql
   SELECT proname FROM pg_proc WHERE proname = 'ensure_weekly_plan_exists';
   ```

3. Usuário tem skill profile?
   ```sql
   SELECT * FROM user_skill_profile WHERE user_id = 'user-uuid';
   ```

4. Logs do edge function mostram erro?

---

### Problema: Erro "User skill profile not found"

**Causa**: Usuário ainda não fez speaking task

**Solução**: 
1. Fazer primeiro speaking task
2. Verificar se profile foi criado:
   ```sql
   SELECT * FROM user_skill_profile WHERE user_id = 'user-uuid';
   ```

---

### Problema: Loading infinito

**Causa**: Edge function timeout ou erro

**Solução**:
1. Verificar logs do edge function
2. Verificar se OpenAI API key está configurada
3. Tentar pull-to-refresh na tela

---

### Problema: Múltiplos planos criados

**Causa**: Race condition (improvável, mas possível)

**Verificar**:
```sql
SELECT 
  id,
  week_start,
  status,
  created_at
FROM weekly_plans
WHERE user_id = 'user-uuid'
ORDER BY created_at DESC;
```

**Solução**: Edge function já tem proteção, mas se acontecer:
```sql
-- Manter apenas o mais recente
UPDATE weekly_plans
SET status = 'archived'
WHERE user_id = 'user-uuid'
  AND status = 'active'
  AND id != (
    SELECT id FROM weekly_plans
    WHERE user_id = 'user-uuid' AND status = 'active'
    ORDER BY created_at DESC
    LIMIT 1
  );
```
